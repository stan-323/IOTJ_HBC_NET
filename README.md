# iotj_chest Chest/Torso HFSS Project

This repository is the clean chest/torso project extracted from the prior 0515 revision workspace.
It keeps the torso HFSS model package and the Python experiment pipeline that consumes the
torso library. Manuscript drafts, compiled PDFs, review notes, work orders, and generated
experiment output directories are intentionally excluded from this public code/model release.

## Layout

- `hfss/torso_main/`
  - `scripts/`: PyAEDT build/export pipeline and HFSS postprocessing.
  - `outputs/`: torso scheduler library, raw S-parameter exports, sampled field proxies, and summary tables.
  - `projects/hbc_torso_main.aedt`: retained AEDT model. Solver caches such as `.aedtresults/` and `.pyaedt/` are intentionally excluded.
- `main_experiment/`
  - `Experiment_results/`: scheduling simulation, baselines, metrics, gates, and plotting code.
  - `tests/`: experiment and LP solver tests.
  - `build_iter10_adaptive.py`: adaptive robustness experiment driver.
- `scripts/rebuild_torso_manuscript_assets.py`: optional figure/table rebuild helper for local
  manuscript workflows. It expects experiment outputs generated by `main_experiment/Experiment_results/run.py`.
- `tests/`: torso HFSS plan and postprocessing tests.

## Release Scope

Uploaded:

- Source code for the scheduler experiments and HFSS postprocessing.
- The compact torso scheduler library, S-parameter exports, field samples, setup/material tables,
  and the retained AEDT model.
- Unit tests and lightweight validation fixtures.

Not uploaded:

- Raw `*_per_slot.csv` traces and generated `output_0515_chest_*` directories.
- Manuscript TeX/PDF files, generated figures, bibliography drafts, and translated drafts.
- Review-response notes, work orders, scratch directories, solver caches, and Python caches.

## Default HFSS Library

The experiment code defaults to the chest/torso scheduler library:

```powershell
hfss\torso_main\outputs\torso_scheduler_library.csv
```

You can still override the library with:

```powershell
$env:IOTJ_HFSS_SCHED_LIBRARY_PATH=(Resolve-Path hfss\torso_main\outputs\torso_scheduler_library.csv)
```

## Setup

```powershell
python -m pip install -r requirements.txt
```

Install `ansys-aedt-core` only on machines that need to build or solve HFSS projects through PyAEDT.

## Verification

Run the torso HFSS package tests from the project root:

```powershell
python -m pytest tests -q
```

Run the main experiment tests from `main_experiment`:

```powershell
cd E:\Codex_Project\iotj_chest\main_experiment
python -m pytest tests -q
```

Regenerate HFSS manifests into a scratch directory:

```powershell
python hfss\torso_main\scripts\run_torso_hfss.py --manifest-only --out-dir scratch\hfss_manifest
```

Postprocess existing torso exports into a scratch directory:

```powershell
python hfss\torso_main\scripts\postprocess_torso_outputs.py --input-dir hfss\torso_main\outputs --out-dir scratch\hfss_postprocess
```

Generated experiment outputs are intentionally not tracked. Recreate them locally with the
drivers in `main_experiment/Experiment_results/run.py` when full manuscript figures or raw
per-slot traces are needed.
